## 项目概述
本项目是一个基于前后端分离架构的视频分享平台，旨在提供视频上传、播放、分类管理、用户互动等核心功能。这个项目的功能我是对着b站的功能做的，当然肯定还有不少b站的功能没复刻上去。不过作为一个视频播放网站的功能还是都有的。
## 核心功能模块
- 视频播放与管理 
- 分类浏览 
- 搜索功能以及根据视频的标签推荐相应视频
- 用户系统 
- 互动功能 
- 后台管理
- 分类轮播图
- 弹幕功能
- 用户聊天功能
- 动态功能
- 对视频的评论
- 视频的点赞，投币，收藏
- 观看历史
- 整理视频合集
- 视频投稿
## 技术栈
### 前端技术
- Vue 3 ：使用Composition API，代码组织更清晰
- Pinia ：轻量级状态管理，替代Vuex，API更简洁
- Vue Router ：前端路由管理，支持嵌套路由和路由守卫
- Element Plus ：UI组件库，界面美观，组件丰富
- Axios ：封装为Request工具，处理HTTP请求，支持拦截器
- Vite ：构建工具，开发时热更新快，生产构建优化好
### 后端技术
- Java 8+ ：稳定可靠的后端开发语言
- Spring Boot ：快速构建Java应用，自动配置省了很多麻烦
- MySQL ：关系型数据库，存储用户数据和视频信息
- Elasticsearch 7.17.x ：搜索引擎，实现高效的视频搜索和智能推荐
- 七牛云SDK ：处理文件上传和存储，CDN加速访问
- Maven ：项目构建和依赖管理
- redis : 用户快速访问
### 前后端连接技术
- RESTful API ：标准化的接口设计，前后端沟通顺畅
- JSON ：数据交换格式，轻量易解析
- Axios封装 ：前端统一处理API请求，包含错误处理和拦截器
- LocalStorage ：前端存储用户登录状态，实现会话保持
- CORS ：后端配置跨域，允许前端跨域请求
  ## 部署指南
### 前端部署
1. 环境准备 ：
   
   - Node.js 14.0+
   - npm 6.0+
2. 安装步骤 ：
   
   ```
   # 进入前端目录
   cd ceshi
   
   # 安装依赖
   npm install
   
   # 开发环境运行
   npm run dev
   
   # 生产环境构建
   npm run build
   
   # 将dist目录部署到Web服务器
   ```
### 后端部署
1. 环境准备 ：
   
   - JDK 1.8+
   - MySQL 5.7+
   - Elasticsearch 7.17.x
   - Maven 3.6+
2. 安装步骤 ：
   
   ```
   # 进入后端目录
   cd shipin
   
   # 构建项目
   mvn clean package
   
   # 运行应用
   java -jar target/shipin-0.0.1-SNAPSHOT.jar
   ```
3. 数据库配置 ：
   
   - 创建MySQL数据库
   - 执行数据库初始化脚本
   - 配置 application.properties 中的数据库连接信息
4. Elasticsearch配置 ：
   
   - 安装并启动Elasticsearch服务
   - 创建视频索引，设置字段映射
   - 配置ES连接信息
5. 七牛云配置 ：
   
   - 注册七牛云账号，创建存储桶
   - 配置Access Key和Secret Key
   - 设置存储桶区域（如华南区域）
## 注意事项
1. 七牛云配置 ：
   
   - 注意区域选择，不同区域的上传地址不同，选错了会导致上传失败
   - 私有存储桶需要使用签名URL访问，后端要处理好签名生成
   - 定期检查存储空间使用情况，避免超出配额
2. Elasticsearch ：
   
   - 安装时注意版本匹配，7.17.x比较稳定
   - 配置文件要调整内存设置，避免内存不足
   - 视频新增或修改时，要及时更新ES索引，确保搜索结果准确
3. 前端优化 ：
   
   - 图片要压缩，不然加载慢影响体验
   - 使用路由懒加载，减少初始加载时间
   - 合理使用缓存，减少重复请求
4. 后端优化 ：
   
   - 数据库查询要加索引，特别是经常搜索的字段
   - 视频上传等耗时操作要使用异步处理
   - 配置合理的连接池，减少连接创建开销
5. 部署问题 ：
   
   - 前后端分离部署时，要注意API地址配置
   - 服务器要选够配置，视频多了的话内存和带宽要跟上
   - 定期备份数据库和重要文件，防止数据丢失
6. 安全措施 ：
   
   - 用户密码要加密存储
   - API接口要做权限控制
   - 防止SQL注入和XSS攻击
   - 上传文件要验证类型和大小，防止恶意文件
